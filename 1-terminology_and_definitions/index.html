JavaScript - Terminology & Definitions

  1. Why is it important to write clean code?
    • Clean code is important because it makes it easier to understand, maintain, and modify the codebase.
  2. What is the difference between good comments and bad comments?
    • Good comments provide meaningful context, explanation, and clarification of the code, and are written in a clear and concise manner.
      On the other hand, bad comments are redundant, outdated, or irrelevant, and do not add any value to the code.
  3. What is an array?
    • An array is a data structure in programming that holds an ordered collection of elements, which can be of any data type such as numbers, strings, objects, etc.
  4. What are arrays useful for?
    • Arrays are useful for storing collections of data of the same or different data types in a single variable.
  5. How do you access an array element?
    • An array element can be accessed by its index number. The index number of an array starts from 0 for the first element, and goes up to n-1, where n is the total number of elements in the array.
  6. How do you change an array element?
    • You can change an array element by simply assigning a new value to a specific index of the array.
  7. What are some useful array properties?
    • length: returns the number of elements in the array.
      indexOf(): returns the first index at which a given element can be found, or -1 if it is not present in the array.
      join(): concatenates all elements of an array into a string and returns it.
      slice(): extracts a portion of an array and returns it as a new array.
      sort(): sorts the elements of an array in ascending or descending order.
      reverse(): reverses the order of elements in an array.
      map(): creates a new array with the results of calling a provided function on every element in the original array.
      reduce(): applies a function against an accumulator and each element in the array (from left to right) to reduce it to a single value.
  8. What are some useful array methods?
    • push() - adds an element to the end of an array.
      pop() - removes an element from the end of an array.
      unshift() - adds an element to the beginning of an array.
      shift() - removes an element from the beginning of an array.
      splice() - allows adding or removing elements from any position in an array.
  9. What are loops useful for?
    • Loops are useful for repeating a set of operations multiple times, either a specified number of times or until a certain condition is met. They are used to automate repetitive tasks and can be used to iterate over arrays, objects, or any other data structure in order to perform a certain operation on each element.
  10. What is the break statement?
    • The break statement is a control flow statement used to immediately exit or break out of a loop, bypassing the normal loop condition and execution of the remaining loop statements.
  11. What is the continue statement?
    • The continue statement is used to skip over the current iteration of a loop and move on to the next iteration. In other words, it skips the rest of the code in the current iteration of the loop and jumps to the next iteration.
  12. What is the DOM?
    • The Document Object Model (DOM) is a programming interface for HTML and XML documents. It represents the structure of a document as a tree-like hierarchy of nodes, with each node representing an element, attribute, or text content. The DOM allows developers to programmatically manipulate and access the content, structure, and style of a document, enabling the creation of dynamic, interactive web pages.
  13. How do you target the nodes you want to work with?
    • The DOM (Document Object Model) is a hierarchical representation of a web page as a tree-like structure, where each element on the page is represented as a node. To target the nodes you want to work with in the DOM, you can use a variety of methods including:
      Using selectors such as CSS selectors, to select specific elements based on their class, id, tag name, etc.
      Using JavaScript properties such as getElementById(), getElementsByClassName(), getElementsByTagName() and querySelector() to select elements based on their id, class name, tag name or a combination of these.
      Traversing the DOM tree and accessing the parent, child, and sibling nodes of elements, to find the elements you want to work with.
  14. How do you create an element in the DOM?
    • To create an element in the DOM, you can use JavaScript to create a new element object and then append it to an existing element using methods like document.createElement and appendChild.
  15. How do you add an element to the DOM?
    • innerHTML property - You can add elements to the DOM by modifying the innerHTML property of a node to include the desired element.
      createElement and appendChild methods - The createElement method is used to create a new element, while the appendChild method is used to append that element as a child of a specified node.
      insertAdjacentHTML method - This method allows you to insert HTML into a node at a specified position, such as beforebegin, afterbegin, beforeend, or afterend.
      createTextNode and appendChild methods - This method allows you to create a text node and add it to the DOM using the appendChild method.
  16. How do you remove an element from the DOM?
    • You can remove an element from the DOM using the removeChild() method or by using the remove() method directly on the element you want to remove.
  17. How can you alter an element in the DOM?
    • To alter an element in the DOM, you can modify its properties, such as its innerHTML, style, or attributes.
  18. When adding text to a DOM element, should you use textContent or innerHTML?
    • if you want to add plain text to an element, use textContent. If you want to add HTML content, use innerHTML.
  19. Where should you include your JavaScript tag in your HTML file when working with DOM nodes?
    • it is recommended to include the JavaScript tag at the end of the HTML file, just before the closing tag. This way, the entire HTML document has finished loading, allowing all the DOM nodes to be fully accessible, before executing any JavaScript code that manipulates the DOM. This helps ensure that the JavaScript code runs smoothly and prevents any errors from occurring due to non-existent DOM nodes.
  20. How do “events” and “listeners” work?
    • "Events" are actions or occurrences that happen in the browser, such as a user clicking a button or a page finishing loading. "Listeners" are functions that wait for a specific event to occur and then respond to it by executing some code.
  21. What are the benefits of using named functions in your listeners?
    • Using named functions as listeners in JavaScript provides better organization and maintainability of code, making it easier to understand and modify.
  22. How do you attach listeners to groups of nodes?
    • You can use a loop to iterate over each node in the group and attach the event listener to each node. You can access the group of nodes using methods such as getElementsByClassName, getElementsByTagName, or querySelectorAll.
  23. What is the difference between the return values of querySelector and querySelectorAll?
    • QuerySelector returns the first element that matches a given CSS selector as a single Element object, while querySelectorAll returns a NodeList of all elements that match the selector.
  24. What does a “nodelist” contain?
    • A node list in JavaScript is a collection of nodes, typically elements, returned by a method such as querySelectorAll. It's an object that resembles an array, but is not an actual array and lacks some of the methods and properties that an array has. A node list can be converted to an array if needed, so that it can be worked with more efficiently.
  25. Explain the difference between “capture” and “bubbling”.
    • Capture refers to the phase in the event flow where events are first captured by the outermost elements and then propagated towards the innermost elements. In other words, the event is first captured by the parent element and then travels to its children.
      Bubbling, on the other hand, refers to the phase in the event flow where events are first handled by the innermost elements and then propagated back up to the outermost elements. In other words, the event is first handled by the child element and then bubbles up to its parent.
  26. What is the difference between objects and arrays?
    • Arrays are ordered lists of values and are accessed by index. They are ideal for storing collections of similar values such as a list of numbers or strings.
      Objects are unordered collections of key-value pairs and are accessed by keys instead of indices. They are ideal for storing more complex data structures, such as collections of related information like a person's name, age, and address.
  27. How do you access object properties?
    • you can access object properties using either the dot notation (obj.property) or bracket notation (obj["property"]).
      The dot notation is simpler and easier to read, but it only works if the property name is a valid identifier (starts with a letter, followed by letters, digits, or underscores).
      The bracket notation allows you to use any string as a property name, including names that are not valid identifiers, by enclosing the property name in quotes.
